git branch 
    verificamos en que ramas estamos trabajando 

creacion de nueva rama e moverse ahi 
    git checkout -b rama 
    git brank (para verificar si estas ahi)
    (una ves estado en la primer rama uno puede trabajar )

# 1. Antes de hacer merge, actualizar tu main local
git checkout main
git pull origin main

# 2. Volver a tu rama y traer los cambios de main
git checkout mi-rama
git merge main
# O mejor: git rebase main (historia más limpia)

# 3. Resolver conflictos si los hay (en tu rama)
# Editar los archivos con conflictos, luego:
git add .
git commit -m "Resolver conflictos"

# 4. Ahora sí, hacer merge a main
git checkout main
git merge mi-rama

# 5. Subir los cambios al repositorio remoto
git push origin main

# 6. Volver a tu rama
git checkout mi-rama
